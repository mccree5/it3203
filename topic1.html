Semantic HTML & Accessibility

Semantic HTML gives meaning to content and creates a clear document outline for users, screen readers, and search engines. Accessibility (a11y) builds on semantics to ensure everyone can use your site regardless of ability, device, or context.

Why Semantic Elements Matter

Structure

<header>, <nav>, <main>, <section>, <article>, <aside>, <footer> create a navigable outline.

Accessibility

Screen readers rely on landmarks and headings. Good semantics reduces ARIA you need to add.

SEO

Search engines infer importance from headings, lists, and landmarks — semantics help them help you.

ARIA: When (and When Not) to Use It

  • Use native elements first. Prefer <button> over clickable <div>.
  • Role parity: Don’t add roles to elements that already have implicit roles (e.g., <nav>).
  • Labels: Use aria-label or aria-labelledby when a visible label is missing.

Quick A11y Checklist

  1. One <h1> per page; use descending headings for structure.
  2. Every image has a purposeful alt (or empty alt for decorative images).
  3. Links are descriptive (“View syllabus” not “Click here”).
  4. Focusable controls show a visible focus ring.
  5. Color contrast passes WCAG AA where possible.
